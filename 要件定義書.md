# ポートフォリオサイト 要件定義書

## 1. プロジェクト概要

### 1.1 目的
個人のスキル・実績を効果的に伝える静的ポートフォリオサイトを構築し、潜在的なクライアントや採用担当者に技術力と問題解決能力をアピールする。

### 1.2 プロジェクト名
Personal Portfolio Website

### 1.3 対象ユーザー
- 採用担当者
- 潜在的なクライアント
- 技術コミュニティのメンバー
- ビジネスパートナー候補

## 2. 機能要件

### 2.1 必須機能

#### 2.1.1 トップページ（ヒーローセクション）
- 自己紹介（名前、肩書き）
- キャッチコピー
- プロフィール写真
- CTAボタン（例：「プロジェクトを見る」「お問い合わせ」）

#### 2.1.2 プロフィールセクション
- 自己紹介文
- 経歴の概要
- 専門分野・得意領域

#### 2.1.3 スキルセクション
- 技術スタック一覧
  - プログラミング言語
  - フレームワーク・ライブラリ
  - インフラ・ツール
  - その他の技術
- 視覚的な表現（アイコン、グラフ、バッジなど）

#### 2.1.4 プロジェクトセクション
各プロジェクトには以下の情報を含む：
- プロジェクト名
- プロジェクトの説明
- **使用技術**（明示的に表示）
- **解決した課題・問題**（明示的に表示）
- 成果・結果
- スクリーンショット・画像
- GitHubリンク（該当する場合）
- デモサイトリンク（該当する場合）
- 開発期間
- 自分の役割

#### 2.1.5 経歴セクション
- 職歴（会社名、期間、役職、業務内容）
- 学歴
- 資格・認定

#### 2.1.6 お問い合わせセクション
- SNSリンク（GitHub、LinkedIn、Twitter/Xなど）
- メールアドレス
- ※お問い合わせフォームは静的サイトのため、外部サービス連携（後述）

#### 2.1.7 フッター
- コピーライト表記
- SNSアイコンリンク
- サイトマップリンク

### 2.2 推奨機能

#### 2.2.1 ブログ・記事セクション（オプション）
- 技術記事へのリンク
- 外部ブログへのリンク集

#### 2.2.2 ダークモード切り替え
- ユーザーがライト/ダークテーマを選択可能

#### 2.2.3 多言語対応（日本語・英語）
- 国際的な機会を広げるため

#### 2.2.4 アニメーション効果
- スクロールアニメーション
- ホバーエフェクト
- ページ遷移効果

## 3. 非機能要件

### 3.1 パフォーマンス
- ページ読み込み時間：3秒以内
- Lighthouse スコア：90点以上（Performance, Accessibility, Best Practices, SEO）
- 画像の最適化（WebP形式、遅延読み込み）
- コードの最小化（minification）

### 3.2 レスポンシブデザイン
- モバイルファースト設計
- 対応デバイス：
  - スマートフォン（320px〜）
  - タブレット（768px〜）
  - デスクトップ（1024px〜）
  - 大画面デスクトップ（1440px〜）

### 3.3 ブラウザ対応
- Chrome（最新版 & 1つ前のバージョン）
- Safari（最新版 & 1つ前のバージョン）
- Firefox（最新版 & 1つ前のバージョン）
- Edge（最新版 & 1つ前のバージョン）

### 3.4 SEO対策
- 適切なメタタグ設定（title, description, keywords）
- OGP（Open Graph Protocol）設定
- Twitter Card設定
- セマンティックHTML使用
- sitemap.xml生成
- robots.txt設置
- 構造化データ（Schema.org）

### 3.5 アクセシビリティ
- WCAG 2.1 レベルAA準拠
- 適切なalt属性
- キーボードナビゲーション対応
- 適切なコントラスト比
- スクリーンリーダー対応

### 3.6 セキュリティ
- HTTPS通信（Firebase Hostingのデフォルト機能）
- セキュリティヘッダーの設定
- XSS対策
- 外部リンクのrel="noopener noreferrer"

## 4. 技術要件

### 4.1 採用技術スタック（確定）

#### 4.1.1 フロントエンドフレームワーク
- **Astro** 4.x（確定）
  - 理由：超高速、SEO最適、静的サイト生成に特化
  - ゼロJavaScriptデフォルト（必要な箇所のみJSを配信）
  - コンポーネントアイランドアーキテクチャ

#### 4.1.2 スタイリング
- **Tailwind CSS** 3.x
  - ユーティリティファーストCSS
  - 高速な開発体験
  - プロダクションビルドで未使用CSSを自動削除

#### 4.1.3 その他の技術
- TypeScript（型安全性）
- React（必要な場合の対話的コンポーネント用）
- View Transitions API（ページ遷移アニメーション）
- CSS Variables（テーマカラー管理）

### 4.2 ホスティング
- **Firebase Hosting**（必須）
  - 無料枠の活用
  - 自動SSL証明書
  - CDN配信
  - カスタムドメイン対応

### 4.3 外部サービス連携

#### お問い合わせフォーム
- **Formspree**（無料プラン）
- **Google Forms**
- **Netlify Forms**（Netlifyも併用する場合）
- **EmailJS**

#### アナリティクス
- Google Analytics 4
- または Vercel Analytics（軽量）

### 4.4 開発環境
- Node.js（v18以上推奨）
- Git / GitHub
- エディタ：VS Code推奨

## 5. コスト要件

### 5.1 ホスティングコスト
- Firebase Hosting無料枠：
  - 10GB/月のストレージ
  - 360MB/日の転送量
  - 想定：**月額 ¥0**（無料枠内）

### 5.2 ドメインコスト（オプション）
- カスタムドメイン：¥1,000〜¥2,000/年
- Firebase Hostingの無料サブドメイン使用なら無料

### 5.3 その他
- 画像・アイコン：無料素材サイト利用
- フォント：Google Fonts（無料）

### 5.4 総コスト見積もり
**月額 ¥0〜¥200**（カスタムドメインを除く）

## 6. デザイン要件

### 6.1 デザインモックアップの作成方法

#### 6.1.1 推奨アプローチ（3つのオプション）

**オプションA：デザインツールを使用（推奨度：★★★）**
- **Figma**（無料プラン）
  - ブラウザベースで使いやすい
  - コミュニティテンプレートが豊富
  - 共有・プレゼンテーションが容易
  - 推奨テンプレート検索：「Portfolio」「Personal Website」
  
**オプションB：既存テンプレートを参考にスキップ（推奨度：★★☆）**
- デザインモックアップをスキップして直接コーディング
- 参考サイトを見ながら実装
- 推奨参考サイト：
  - [Astro Theme Showcase](https://astro.build/themes/)
  - Dribbble、Behanceでポートフォリオ検索
  - 開発者ポートフォリオのベストプラクティス例

**オプションC：紙ベースのワイヤーフレーム（推奨度：★☆☆）**
- 手書きで簡単なレイアウトスケッチ
- セクション配置と情報の流れを確認
- 最小限の時間投資

#### 6.1.2 推奨：Figmaでの作成手順
1. Figmaアカウント作成（無料）
2. コミュニティから「Portfolio Template」を検索
3. テンプレートを複製して自分用にカスタマイズ
4. デスクトップ（1440px）とモバイル（375px）の2画面を作成
5. カラースキーム、フォント、レイアウトを決定

#### 6.1.3 デザイン作成の時間配分
- Figma使用：2〜4時間
- テンプレート参考＋直接実装：0時間（設計フェーズをスキップ）
- 紙スケッチ：30分〜1時間

**結論：時間を節約したい場合は、Astroのテーマを参考にしながら直接実装することを推奨します。**

### 6.2 デザインコンセプト
- **プロフェッショナル**かつ**親しみやすい**
- **クリーン**で**モダン**
- 技術力を感じさせるデザイン
- 情報の可読性を重視
- **ダークモード対応**（トレンド）

### 6.3 カラースキーム（推奨例）
- プライマリカラー：`#3B82F6`（Blue-500）
- セカンダリカラー：`#8B5CF6`（Purple-500）
- 背景色（ライト）：`#FFFFFF` / `#F9FAFB`
- 背景色（ダーク）：`#111827` / `#1F2937`
- テキストカラー（ライト）：`#111827`
- テキストカラー（ダーク）：`#F9FAFB`
- アクセントカラー：`#10B981`（Emerald-500、CTAボタン用）

### 6.4 タイポグラフィ
- 見出し用フォント：**Inter**（モダン、視認性高い）
- 本文用フォント：**Inter**（統一感）
- 日本語フォント：**Noto Sans JP**
- フォントサイズ：
  - H1: 3rem (48px)
  - H2: 2.25rem (36px)
  - H3: 1.5rem (24px)
  - Body: 1rem (16px)

### 6.5 レイアウト
- **シングルページ構成**（SPA風）
- スムーススクロール
- 固定ヘッダー（ナビゲーション、透過効果）
- セクションごとに明確な区切り
- コンテナ最大幅：1280px

### 6.6 画像・アセット
- プロフィール写真：高品質（最低500x500px）
- プロジェクト画像：統一感のあるサイズ（推奨：1200x630px）
- アイコン：**Lucide Icons**（Astroと相性良い）または **Heroicons**
- ファビコン：複数サイズ用意（16x16, 32x32, 180x180, 512x512）

## 7. プロジェクトセクションの詳細要件

### 7.1 必須表示項目
各プロジェクトカードには以下を**必ず明示**：

1. **使用技術**
   - 表示方法：バッジ、タグ、アイコン付きリスト
   - 例：React, TypeScript, Firebase, Tailwind CSS

2. **解決した課題**
   - 表示方法：簡潔な文章（2〜3文）
   - 例：
     - 「手作業で行っていたデータ集計作業を自動化し、業務時間を70%削減」
     - 「ユーザーの離脱率が高かった申込フローを再設計し、コンバージョン率を2倍に改善」
     - 「レガシーシステムのモダナイゼーションにより、保守コストを50%削減」

3. **成果・インパクト**
   - 定量的な結果があれば記載
   - 例：「処理速度3倍向上」「ユーザー数1000人突破」

### 7.2 プロジェクトカードのUI
- カード形式での表示
- ホバーエフェクト
- クリックで詳細モーダル表示（オプション）
- フィルタリング機能（技術タグでフィルタ）
